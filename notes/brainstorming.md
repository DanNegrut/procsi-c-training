This is a camp that runs two weeks, for undergrads students from the California State System. They are computer novices. I want them to get some exposure to things that are relevant but likely unknown to them.

Topics i want them the learn a bit aobut:
- the command line and bash
- learn about git and GitHub, the very basics
- the very basics of Python - should i go pip or conda? Choose the easiest and most painless path
- introduce them to claude
- how to render an image in Blender
- how to generate a mechanism in SolidWOrks, and then how to import assets to have it rendered in Blender
- what it means to "simulate"
    - have a block of mass $m$, push by a force $f$, over 10 seconds.
    - write the equation of motion
    - get an idea about what it means to apply forward Euler to find how the block moves over time
    - take this lesson to generate a movie of the sliding block by using their Blender knowledge (have some assets in the back, to see how the block slides; can't have a white canvas)
- connect the example with the moving block to the idea of simulation
- introduce them to PyChrono
    - have them install PyChrono
    - have them run a couple of nice sims that have practical relevance (which ones?)
- explain what RAG means, and have them use chrono-rag to generate some PyChrono models
